# CCTV Video Processing System

Система для автоматической синхронизации и нарезки видеофрагментов с камер наблюдения (NVR/серверов) по заданным интервалам.

## 1. Требования

- Python 3.10+
- FFmpeg (должен быть установлен в системе или указан путь в `.env`)

## 2. Настройка (.env)

Создайте файл `.env` в корне проекта (если его нет) и укажите параметры:

```env
REMOTE_HOST=local          # 'local' для локальной папки или IP сервера для SSH/SFTP
REMOTE_PORT=22             # Порт SSH (если используется удаленный сервер)
REMOTE_USER=user           # Логин SSH
REMOTE_PASSWORD=password   # Пароль SSH
REMOTE_BASE_PATH=videos    # Папка, где лежат исходные видео
```

### Важно: Структура папок с видео
Система ожидает, что внутри `REMOTE_BASE_PATH` видео разложены по датам и комнатам:
`videos / ГГГГ-ММ-ДД / ПАПКА_КОМНАТЫ / файлы.mp4`

**Пример:**
`videos/2026-02-18/504 / 504-2_start_20260218135939.mp4`

## 3. Как система узнает комнату из файла?

Программа берет самое начало имени файла и останавливается перед первым символом `-` (тире) или `_` (подчеркивание).

**Примеры:**
- `504-2_v_bino...` -> комната **504**
- `608_1_hona...` -> комната **608**
- `room1_12-00.mp4` -> комната **room1**

Это позволяет автоматически объединять видео с нескольких камер одной комнаты (например, 504-1 и 504-2), если вы укажете в задаче просто номер `504`.

## 4. Запуск сервера

```powershell
python -m app.main
```
После запуска Swagger UI будет доступен по адресу: `http://127.0.0.1:8000/docs`

## 5. Как тестировать через Swagger

### Шаг 1: Создание задачи на нарезку
Используйте эндпоинт **POST `/process-day`**.

#### Как нарезать несколько комнат или интервалов сразу:
Вы можете указать список комнат и список временных интервалов в одном запросе. Система создаст отдельную задачу для каждой комнаты, и в каждой из них нарежет все указанные интервалы.

**Пример (несколько комнат и интервалов):**
```json
{
  "date": "2026-02-18",
  "rooms": ["504", "608"],
  "intervals": [
    {
      "start": "13:30",
      "end": "14:00"
    },
    {
      "start": "14:10",
      "end": "14:20"
    }
  ]
}
```
*В данном примере система создаст 2 задачи (по одной на комнату) и в каждой нарежет по 2 видеофайла.*

#### Как нарезать по отдельности:
Если вам нужна только одна конкретная комната, просто укажите её одну в списке: `"rooms": ["504-2"]`.

### Шаг 2: Проверка статуса
Используйте эндпоинт **GET `/status/{date}`**.
Введите дату (например, `2026-02-18`) и нажмите Execute. Вы увидите логи процесса и текущий статус (`processing` или `completed`).

### Шаг 3: Получение списка нарезанных видео
Используйте эндпоинт **GET `/videos/{date}/{room}`**.
Он вернет список имен файлов, которые уже готовы.

### Шаг 4: Скачивание результата
**Внимание!** Не используйте кнопку "Execute" в Swagger для скачивания больших файлов (браузер может зависнуть). 
Используйте прямую ссылку в адресной строке браузера:
`http://127.0.0.1:8000/download/{date}/{room}/{filename}`

**Пример:**
`http://127.0.0.1:8000/download/2026-02-18/504-2/2026-02-18_504-2_14-10_14-20.mp4`

## 6. Места хранения данных

- `storage/downloads/` — временная папка для оригинальных видео.
- `storage/output/` — **здесь лежат готовые нарезанные фрагменты**.
- `videos.db` — база данных SQLite со списком задач и логами.
- `app.log` — подробные логи работы системы.
